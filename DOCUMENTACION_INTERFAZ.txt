1. LIBRERÍAS Y MÓDULOS IMPORTADOS
================================================================================

1.1 ttkbootstrap (ttk)
   - ¿Qué es?: Librería para crear interfaces gráficas modernas en Python
   - ¿Para qué se usa?: Crear ventanas, botones, etiquetas, frames y otros
     elementos visuales del juego
   - Ejemplos de uso:
     * ttk.Window() - Crea la ventana principal
     * ttk.Button() - Crea botones
     * ttk.Label() - Crea etiquetas de texto
     * ttk.Frame() - Crea contenedores para organizar elementos
     * ttk.Entry() - Crea campos de texto para entrada
     * ttk.Combobox() - Crea listas desplegables
     * ttk.Notebook() - Crea pestañas
     * ttk.ScrolledText() - Crea áreas de texto con scroll

1.2 ttkbootstrap.constants (BOTH, LEFT, RIGHT, etc.)
   - ¿Qué es?: Constantes que definen posiciones y comportamientos
   - ¿Para qué se usa?: Especificar cómo se deben colocar los elementos
   - Ejemplos:
     * BOTH - Hace que un elemento se expanda en ambas direcciones
     * LEFT/RIGHT - Alinea elementos a la izquierda o derecha
     * WORD - Hace que el texto se ajuste por palabras

1.3 ttkbootstrap.widgets.Meter
   - ¿Qué es?: Widget que muestra barras de progreso circulares o semicirculares
   - ¿Para qué se usa?: Mostrar vida, mana y otras estadísticas del personaje
   - Ejemplo: Meter para mostrar la vida del jugador

1.4 ttkbootstrap.widgets.tableview.Tableview
   - ¿Qué es?: Widget para crear tablas con datos
   - ¿Para qué se usa?: Mostrar habilidades en formato de tabla

1.5 PIL (Pillow) - Image, ImageTk
   - ¿Qué es?: Librería para trabajar con imágenes
   - ¿Para qué se usa?: Aunque está importada, actualmente no se usa en el código
   - Nota: Se podría usar para mostrar imágenes de personajes o enemigos

1.6 os (módulo estándar de Python)
   - ¿Qué es?: Módulo para interactuar con el sistema operativo
   - ¿Para qué se usa?: Verificar si existen archivos, rutas, etc.
   - Nota: Aunque está importado, se usa principalmente en otros módulos

1.7 random (módulo estándar de Python)
   - ¿Qué es?: Módulo para generar números aleatorios
   - ¿Para qué se usa?: Seleccionar enemigos aleatorios para las batallas

2. ESTRUCTURA DE DATOS PRINCIPAL

2.1 estado_interfaz (diccionario global)
   - ¿Qué es?: Un diccionario que almacena todos los elementos de la interfaz
   - ¿Para qué se usa?: Guardar referencias a widgets y datos del juego
   - Contiene:

     * "container" - Contenedor principal de la interfaz
     * "frame_actual" - El frame que se está mostrando actualmente

     * "entry_nombre" - Campo de texto para el nombre del personaje
     * "clase_seleccionada" - Variable que guarda la clase seleccionada
     * "meter_vida", "meter_ataque", "meter_defensa" - Barras de estadísticas
     * "notebook" - El contenedor de pestañas
     * "meter_vida_jugador", "meter_mana_jugador" - Barras del jugador
     * "combo_enemigos" - Lista desplegable de enemigos
     * "texto_log" - Área de texto para el log del combate
     * "tabla_habilidades" - Tabla de habilidades
     * "enemigo_actual" - Diccionario con datos del enemigo actual
     * "meter_vida_combate", "meter_mana_combate" - Barras en combate
     * "meter_vida_enemigo" - Barra de vida del enemigo
     * "frame_menu_habilidades" - Frame del menú de habilidades
     * "frame_menu_objetos" - Frame del menú de objetos

6. MÉTODOS DE WIDGETS UTILIZADOS
================================================================================

6.1 .pack()
   - Coloca el widget en la ventana usando el gestor de geometría pack
   - Parámetros comunes:
     * fill=BOTH - Expande en ambas direcciones
     * expand=True - Permite expansión
     * padx, pady - Espaciado horizontal y vertical
     * side=LEFT/RIGHT - Posición lateral

6.2 .configure()
   - Cambia las propiedades de un widget
   - Uso: Actualizar valores de barras, texto, etc.

6.3 .get()
   - Obtiene el valor de un widget
   - Uso: Obtener texto de Entry, selección de Combobox, etc.

6.4 .insert()
   - Inserta texto en un widget
   - Uso: Poner texto por defecto en Entry

6.5 .destroy()
   - Elimina un widget
   - Uso: Limpiar pantalla antes de mostrar otra

6.6 .select()
   - Selecciona una pestaña
   - Uso: Cambiar a una pestaña específica

7. FLUJO DEL PROGRAMA
================================================================================

1. Se inicializa la interfaz con inicializar_interfaz()
2. Se muestra la pantalla de inicio
3. El usuario crea un personaje
4. Se muestra la pantalla principal con pestañas
5. El usuario puede:
   - Ver su estado
   - Ir a batalla
   - Ver habilidades
   - Ver inventario
6. En batalla, se muestra la pantalla de combate
7. El usuario puede usar habilidades, objetos o defender
8. Después del combate, vuelve a la pantalla principal

9. SOBRE __pycache__
================================================================================

¿Qué es __pycache__?
-------------------
__pycache__ es una carpeta que Python crea automáticamente cuando ejecutas
tu programa. Contiene archivos .pyc (Python Compiled).

¿Por qué se crea?
-----------------
Cuando Python ejecuta un archivo .py, lo convierte a bytecode (código
intermedio) para que se ejecute más rápido. Estos archivos compilados se
guardan en __pycache__ para que la próxima vez que ejecutes el programa,
Python no tenga que compilar todo de nuevo.

¿Es necesario?
--------------
Sí, es parte normal del funcionamiento de Python. Hace que tu programa se
ejecute más rápido en ejecuciones posteriores.

¿Puedo eliminarlo?
------------------
Sí, puedes eliminarlo sin problemas. Python lo volverá a crear cuando
ejecutes el programa de nuevo. No afecta el funcionamiento de tu código.

¿Cómo evitar que se cree?
-------------------------
No es recomendable evitarlo, pero si quieres, puedes:
1. Agregar __pycache__/ a tu archivo .gitignore (si usas Git)
2. Eliminarlo manualmente cuando quieras limpiar el proyecto

¿Qué contiene?
-------------
Archivos .pyc que son versiones compiladas de tus archivos .py:
- personaje.cpython-XX.pyc
- enemigo.cpython-XX.pyc
- combate.cpython-XX.pyc
- interfaz.cpython-XX.pyc

(XX es el número de versión de Python, como 311 para Python 3.11)

10. RESUMEN
================================================================================

interfaz.py es el módulo que maneja toda la interfaz gráfica del juego.
Utiliza ttkbootstrap para crear una interfaz moderna y organiza todo en
funciones (sin POO). Usa un diccionario global (estado_interfaz) para
almacenar referencias a todos los elementos de la interfaz.

El código está estructurado de forma funcional, usando:
- Funciones en lugar de clases
- Diccionarios en lugar de objetos
- Variables globales para compartir estado
- Importaciones dinámicas cuando es necesario
